[계층 분리 순서] / 02-02 폴더의 README 파일 내용임

- 디자인 패턴으로 작업(파일 분리)
  계층구조가 하나의 API를 각각의 기능별로 함수를 분리시켜서
  의존성을 최소화시키고 유지보수가 용이하게 해줌

폴더별로 목적이 존재

- controllers
  - 프론트앤드와 백엔드 사이에 데이터 주고받는 부분(상태코드 포함)의 코드를 기재하는 곳
- routes
  - HTTP 메소드, URI(인터넷 주소)와 컨트롤러 함수를 매칭시키는 코드를 기재하는 곳
- services
  - 컨트롤러에서 데이터를 넘겨 받아서, 실제 비즈니스 로직이 처리되는 코드를 기재하는 곳
- models
  - 데이터베이스에서 데이터를 입력 수정 삭제 조회 하는 등의 코드가 기재되는 곳

아래와 같은 방식으로 동작한다.
main.py
-> routes -> coutrollers -> services -> models

postgres_data 폴더는 캐시 내용 등이 담겨 있어 .gitignore 폴더에 기재해 깃 허브에 커밋 제외 예정
